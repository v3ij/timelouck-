=== Sheet: Modules ===
1. Modules Required to Get the System Working | 
Module | Description
User Management | Registration, profile management, KYC (if needed), biometric enrollment, balance/wallet, time-plan subscriptions.
Device Management | Onboarding of access terminals/kiosks, firmware/software updates, health monitoring, remote configuration, offline rules sync.
Access Control Engine | Core logic for identify & verify (biometric match, RFID validation, QR code decoding), session start/stop, anti-passback, limit enforcement.
Time Engine | Real‑time per‑second session timer, elapsed-time tracking, connection to billing.
Billing Engine | Converts tracked time to cost using configurable rates (per minute, per service/location), supports auto‑stop limits and pre‑paid balance checks.
Payment Connectors | Abstraction layer to integrate with mobile money (e.g., MTN/Airtel Uganda), card processors, bank transfers, internal wallet debits.
Accounting Engine | Double‑entry ledger, daily/monthly revenue reconciliation, financial audit trail, integration with external accounting systems.
Notification Service | SMS, push notifications (Firebase/APNs), email alerts – session start, low balance, auto‑payment, receipts.
Customer Mobile App | Scan QR at entry, view live session timer and live cost, check balance, top‑up, usage history, purchase time plans.
Business Owner Web Dashboard | Real‑time overview (active users, revenue, device status), historical reports, user/demand analytics, system settings.
Reports & Alerts Engine | Scheduled reports (daily/weekly/monthly revenue, usage by location), real‑time alerts (device offline, balance below threshold, suspicious activity).
Audit & Logs | Immutable logs for all access events, payment transactions, admin changes, for security and compliance.
Data Analytics | Aggregated metrics, revenue trends, top services/locations, peak hours, user behavior insights.
Admin Panel | System configuration (rates, locations, plans), user/device management, manual overrides, dispute resolution.
Offline Mode Module | Local terminal cache of authorised users and rules; synchronisation when connectivity resumes.
